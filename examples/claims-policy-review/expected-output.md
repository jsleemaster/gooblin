# Expected Output

## Shipcheck

### Cut unnecessary work

Cut unsupported performance and universal compatibility claims.

### Ground the architecture

Keep the README focused on what Gooblin is: a plugin-style Agent Skills pack that also works as Markdown.

### Reproduce or bound bugs

List which install paths were actually verified and which remain intended but unverified.

### Defer optional scope

Defer benchmark language until there is a documented method, dataset, sample size, limitations, and reproduction path.

### Safety floor

Do not imply safety improvements without evidence.

### Verification

Check README, docs/claims-policy.md, docs/verified-install-paths.md, and release notes for unsupported claims.

### Verdict

Ship only after unsupported claims are removed or converted into measured, cited, reproducible statements.
