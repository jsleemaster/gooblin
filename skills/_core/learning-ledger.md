# Learning Ledger

Gooblin does not implement automatic self-modification.

Reusable lessons should be proposed in a reviewable way, then accepted or rejected by a maintainer.

## Rules

- Do not silently edit skills based on one task.
- Do not store private user data as a lesson.
- Keep lessons general, reusable, and tied to observable evidence.
- Mark every proposed lesson as needing maintainer review.

## Template

```markdown
## Learned

Reusable lesson:
Skill impacted:
Suggested update:
Needs maintainer review: yes
```
