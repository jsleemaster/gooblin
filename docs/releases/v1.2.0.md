# Gooblin v1.2.0 Release Review

## Goal

Expand Gooblin from a minimal skill pack into a broader docs-first plugin pack without adding a runtime.

## Changes

- Added host adapter recipes for Codex, Claude Code, Gemini CLI, OpenCode, Devin, and Hermes Agent.
- Added operations documentation for status checks, configuration, uninstall, and statusline recipes.
- Added dependency-free validation scripts and `npm run validate`.
- Kept adapter claims bounded to documented host behavior or manual recipes.
- Removed over-scoped brand and roadmap docs from main before this release.
- Updated plugin metadata to `1.2.0`.

## Verification

- `npm run validate`
- `git diff --check`
- JSON manifest parse check.
- Hook smoke test.
- `npm pack --dry-run`
- Local Codex plugin install verification.
- Local Claude plugin install verification.
- Codex remote release-branch install verification.
- Post-release Codex remote install verification.
- Post-release Claude remote install verification.
- Local global npm install verification.

## Review Notes

This is a docs-first minor release. It does not add app scaffolding, runtime services, dependencies, CI, benchmark results, telemetry, cloud requirements, or automatic host mutation.
