# Gooblin v1.2.1 Release Review

## Goal

Patch the public skill pack metadata so the maintenance guardrails, issue-derived checks, and lightweight positioning updates after v1.2.0 are included in a tagged release.

## Changes Since v1.2.0

- Added a PR template and maintenance impact matrix for public-surface drift control.
- Added `validate:maintenance` for release notes, command/example links, adapter index links, and maintenance docs.
- Added stale version checks across public install and compatibility docs.
- Added hook smoke validation for quiet, off, disabled, disable, and verbose reminder modes.
- Added a Korean multilingual evaluation fixture and language-preservation rubric rule.
- Documented `/shipcheck` working-tree, branch, and release review scopes with numbered findings.
- Clarified that Gooblin is intentionally small: a lightweight engineering judgment gate, not a full SDLC framework.
- Updated plugin metadata to `1.2.1`.

## Verification

- GitHub API content checks for current main and release metadata.
- Version metadata synchronized across `package.json`, `plugin.yaml`, `.claude-plugin/plugin.json`, `.codex-plugin/plugin.json`, `hooks/claude-codex-hooks.json`, README badge, and operations docs.
- Local `npm run validate` and plugin install checks were not re-run in this restricted filesystem session; repeat them from a writable checkout before making new compatibility claims.

## Review Notes

This is a docs-first patch release. It does not add runtime services, dependencies, CI, benchmark results, telemetry, cloud requirements, statusline automation, or automatic host mutation.
