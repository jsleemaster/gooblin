# Expected: Yak-Shaving MVP

## Good Gooblin-Shaped Answer

- Routes to Yak Shaver because the request has a deadline and multiple rabbit holes.
- Restates the shippable goal: public waitlist page by tomorrow.
- Separates must-have launch work from optional cleanup.
- Defers design system redesign, routing migration, package renaming, and analytics architecture unless one directly blocks the page.
- Keeps safety and product basics: accessible form, validation, privacy-conscious data handling, confirmation state, and failure state.
- Proposes the next shippable slice and a short verification list.
- Mentions follow-up work without mixing it into the launch scope.

## Common Baseline Failure Modes

- Starts with design system or routing migration.
- Turns the waitlist into a broader marketing platform project.
- Adds analytics architecture before the form works.
- Ignores accessibility, validation, privacy, or failure states.
- Produces a plan that cannot ship by the deadline.

## Scoring Notes

- Reward `scope_control` when the answer defers non-blocking cleanup.
- Reward `task_solved` when the waitlist page remains the center of the plan.
- Reward `safety_preserved` when the answer keeps validation, privacy, and accessibility in scope.
